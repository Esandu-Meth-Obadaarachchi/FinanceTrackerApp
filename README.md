# FinTrack — Financial Tracker

A personal finance tracker built with **Flutter** (web + Android) and
**Firebase** (Auth + Cloud Firestore). The UI is a faithful rebuild of the
`Financial Tracker _Standalone_.html` design — dark/light themes, LKR currency,
Sri Lanka tax-aligned categories, and live cloud sync across devices.

## Features

- **Email + password auth** (Firebase Authentication)
- **Dashboard** — total balance, monthly income/expense/net, budget progress,
  expense breakdown, recent activity
- **Transactions** — income / expense / transfer, search, month filter,
  grouped by date, add / edit / delete
- **Accounts** — bank / cash / fixed deposit, computed balances, in/out totals
- **Loans** — money lent & borrowed, mark repaid, running totals
- **Tax Report** — period summary, income/expense breakdown, CSV export
- **Per-user cloud sync** — each user's data lives under `users/{uid}` in
  Firestore and syncs live across web and mobile
- Dark / light theme, mobile-first layout (centred 480px frame on web)

## Prerequisites

- Flutter 3.32.x  ·  Firebase CLI  ·  FlutterFire CLI  ·  Node 18+

## 1 · Firebase setup

```bash
# Re-authenticate (the saved token expires periodically)
firebase login --reauth

# Create the project + wire the app + deploy security rules
./setup-firebase.sh
```

`setup-firebase.sh` creates the Firebase project, runs
`flutterfire configure` (which writes `lib/firebase_options.dart` and
registers the web + Android apps), creates the Firestore database, and
deploys `firestore.rules`.

Then finish **two clicks** in the
[Firebase console](https://console.firebase.google.com/):

1. **Authentication → Get started → Email/Password → Enable → Save**
2. **Firestore Database** → if not auto-created, **Create database →
   Production mode → location → Enable**

## 2 · Run

```bash
flutter run -d chrome      # web
flutter run -d android     # Android device/emulator
```

## 3 · Deploy the web app to Netlify

```bash
flutter build web --release

# Option A — Netlify CLI (no Git needed)
npm i -g netlify-cli
netlify login
netlify deploy --prod --dir=build/web

# Option B — connect this repo in the Netlify UI.
#   netlify.toml + netlify-build.sh install Flutter and build in CI.
```

The SPA redirect (`/* → /index.html`) is configured in `netlify.toml`.

## Data model

```
users/{uid}/accounts/{id}      name, type, colorHex, openingBalance
users/{uid}/transactions/{id}  date, type, accountId, toAccountId,
                               category, note, amount, status
users/{uid}/loans/{id}         loanType, who, amount, reason, date,
                               accountId, status
```

Account balances are **computed** (`openingBalance` + transaction effects),
so edits and deletes stay consistent automatically.
